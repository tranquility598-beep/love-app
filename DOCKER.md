# Использование Love с Docker 🐳

Этот проект поддерживает работу серверной части в Docker-контейнерах. Это удобно для macOS и других платформ, так как позволяет запустить все окружение (сервер + база данных MongoDB) одной командой без необходимости устанавливать MongoDB отдельно.

## Предварительные требования

У вас должен быть установлен:
1. **Docker Desktop** (для Windows или macOS) — [Скачать](https://www.docker.com/products/docker-desktop/)
2. Node.js (для запуска клиента)

## Быстрый старт

1. **Запустите Docker-окружение**:
   Откройте терминал в корневой папке проекта и выполните:
   ```bash
   npm run docker:up
   ```
   *Это скачает образы и запустит сервер и базу данных в фоновом режиме.*

2. **Запустите клиент (Electron)**:
   ```bash
   npm run electron
   ```
   *Клиент автоматически подключится к серверу, работающему в Docker.*

## Полезные команды

*   **Остановить все контейнеры**:
    ```bash
    npm run docker:down
    ```

*   **Пересобрать образы** (если вы изменили код сервера):
    ```bash
    npm run docker:build
    ```

*   **Посмотреть логи сервера**:
    ```bash
    docker logs -f love-server
    ```

## Как это устроено?

*   **MongoDB**: Данные базы сохраняются в разделе `mongo-data`, поэтому они не пропадут при перезагрузке контейнеров.
*   **Загрузки**: Аватарки и файлы, которые вы загружаете, сохраняются в папке `server/uploads` благодаря монтированию разделов (`volumes`).
*   **Порты**: Сервер слушает порт `5555`, база данных — `27017`.

---

**Примечание для macOS**: Docker отлично работает на Mac (Intel и Apple Silicon). Это идеальный способ запустить проект с «чистым» окружением.
